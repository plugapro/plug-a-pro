# WhatsApp Marketing Preference Management — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give customers full control over WhatsApp marketing messages (opt-in/out), enforce that control on every outbound send, and surface preferences in the customer-facing PWA and admin console.

**Architecture:** A new `lib/whatsapp-policy.ts` module acts as the single enforcement gate — all 15 send functions in `lib/whatsapp.ts` call `canSend()` before dispatching. Customer consent state lives on the `Customer` model (6 new fields) with a `WhatsappPreferenceLog` audit table. Opt-out can happen via the PWA, the WhatsApp bot (STOP keywords), or admin override.

**Tech Stack:** Prisma 5 (schema + migrations), Next.js 16 App Router, TypeScript, Supabase auth, existing `lib/whatsapp.ts` + `lib/messaging-templates.ts`

```typescript
// lib/whatsapp-policy.ts
// Central enforcement gate for all outbound WhatsApp messages.
//
// Usage:
//   const check = await canSend(phone, 'booking_reminder')
//   if (!check.allowed) return  // skip silently — caller logs if needed
//
//   await applyOptOut(phone, 'bot')   // STOP keyword in bot
//   await applyOptIn(phone, 'pwa')    // customer enabled toggle

import { db } from './db'
import { TEMPLATE_REGISTRY } from './messaging-templates'

export type PolicyResult =
  | { allowed: true }
  | { allowed: false; reason: 'service_opted_out' | 'marketing_opted_out' | 'customer_not_found' | 'unknown_template' }

/**
 * Check whether a message may be sent to this phone number.
 * Pass the E.164 phone and the exact template name from TEMPLATE_REGISTRY.
 */
export async function canSend(phone: string, templateName: string): Promise<PolicyResult> {
  const tpl = TEMPLATE_REGISTRY.find((t) => t.name === templateName)
  if (!tpl) return { allowed: false, reason: 'unknown_template' }

  // UTILITY messages are blocked only if service opt-in is false
  if (tpl.category === 'UTILITY') {
    const customer = await db.customer.findUnique({
      where: { phone },
      select: { whatsappServiceOptIn: true },
    })
    if (!customer) return { allowed: false, reason: 'customer_not_found' }
    if (!customer.whatsappServiceOptIn) return { allowed: false, reason: 'service_opted_out' }
    return { allowed: true }
  }

  // MARKETING messages require explicit opt-in
  const customer = await db.customer.findUnique({
    where: { phone },
    select: { whatsappMarketingOptIn: true },
  })
  if (!customer) return { allowed: false, reason: 'customer_not_found' }
  if (!customer.whatsappMarketingOptIn) return { allowed: false, reason: 'marketing_opted_out' }
  return { allowed: true }
}

/** Opt a phone number OUT of marketing messages. Creates audit log entry. */
export async function applyOptOut(
  phone: string,
  source: 'bot' | 'pwa' | 'admin' | 'webhook' | 'import',
  opts?: { actorId?: string; note?: string; serviceOptOut?: boolean }
): Promise<void> {
  const customer = await db.customer.findUnique({ where: { phone } })
  if (!customer) return

  const field = opts?.serviceOptOut ? 'whatsappServiceOptIn' : 'whatsappMarketingOptIn'
  const oldValue = opts?.serviceOptOut ? customer.whatsappServiceOptIn : customer.whatsappMarketingOptIn
  const now = new Date()

  await db.$transaction([
    db.customer.update({
      where: { phone },
      data: opts?.serviceOptOut
        ? { whatsappServiceOptIn: false }
        : {
            whatsappMarketingOptIn: false,
            whatsappMarketingOptOutAt: now,
            whatsappMarketingSource: source,
            lastWhatsappPrefSyncAt: now,
          },
    }),
    db.whatsappPreferenceLog.create({
      data: {
        customerId: customer.id,
        field,
        oldValue,
        newValue: false,
        source,
        actorId: opts?.actorId,
        note: opts?.note,
      },
    }),
  ])
}

/** Opt a phone number IN to marketing messages. Creates audit log entry. */
export async function applyOptIn(
  phone: string,
  source: 'bot' | 'pwa' | 'admin' | 'webhook' | 'import',
  opts?: { actorId?: string; note?: string }
): Promise<void> {
  const customer = await db.customer.findUnique({ where: { phone } })
  if (!customer) return

  const oldValue = customer.whatsappMarketingOptIn
  const now = new Date()

  await db.$transaction([
    db.customer.update({
      where: { phone },
      data: {
        whatsappMarketingOptIn: true,
        whatsappMarketingOptInAt: now,
        whatsappMarketingSource: source,
        lastWhatsappPrefSyncAt: now,
      },
    }),
    db.whatsappPreferenceLog.create({
      data: {
        customerId: customer.id,
        field: 'whatsappMarketingOptIn',
        oldValue,
        newValue: true,
        source,
        actorId: opts?.actorId,
        note: opts?.note,
      },
    }),
  ])
}
```

// app/api/customer/preferences/route.ts
import { NextRequest, NextResponse } from 'next/server'
import { createRouteHandlerClient } from '@supabase/auth-helpers-nextjs'
import { cookies } from 'next/headers'
import { db } from '@/lib/db'
import { applyOptIn, applyOptOut } from '@/lib/whatsapp-policy'

export async function GET(req: NextRequest) {
  const supabase = createRouteHandlerClient({ cookies })
  const { data: { session } } = await supabase.auth.getSession()
  if (!session) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })

  const customer = await db.customer.findFirst({
    where: { userId: session.user.id },
    select: {
      whatsappServiceOptIn: true,
      whatsappMarketingOptIn: true,
      whatsappMarketingOptInAt: true,
      whatsappMarketingOptOutAt: true,
    },
  })

  if (!customer) return NextResponse.json({ error: 'Not found' }, { status: 404 })
  return NextResponse.json(customer)
}

export async function PATCH(req: NextRequest) {
  const supabase = createRouteHandlerClient({ cookies })
  const { data: { session } } = await supabase.auth.getSession()
  if (!session) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })

  const customer = await db.customer.findFirst({
    where: { userId: session.user.id },
    select: { phone: true },
  })
  if (!customer) return NextResponse.json({ error: 'Not found' }, { status: 404 })

  const body = await req.json() as { whatsappMarketingOptIn?: boolean }

  if (typeof body.whatsappMarketingOptIn === 'boolean') {
    if (body.whatsappMarketingOptIn) {
      await applyOptIn(customer.phone, 'pwa')
    } else {
      await applyOptOut(customer.phone, 'pwa')
    }
  }

  return NextResponse.json({ ok: true })
}
```

'use client'

import { useEffect, useState } from 'react'

export function WhatsappPreferencesCard() {
  const [marketingOptIn, setMarketingOptIn] = useState<boolean | null>(null)
  const [saving, setSaving] = useState(false)

  useEffect(() => {
    fetch('/api/customer/preferences')
      .then((r) => r.json())
      .then((d) => setMarketingOptIn(d.whatsappMarketingOptIn ?? false))
  }, [])

  async function toggle() {
    if (marketingOptIn === null) return
    setSaving(true)
    const next = !marketingOptIn
    await fetch('/api/customer/preferences', {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ whatsappMarketingOptIn: next }),
    })
    setMarketingOptIn(next)
    setSaving(false)
  }

  if (marketingOptIn === null) return null

  return (
    <div className="rounded-lg border p-4">
      <h3 className="font-semibold text-sm mb-1">WhatsApp Notifications</h3>
      <p className="text-xs text-muted-foreground mb-3">
        Receive special offers and promotions via WhatsApp. You&apos;ll always get
        booking updates regardless of this setting.
      </p>
      <label className="flex items-center gap-2 cursor-pointer select-none">
        <input
          type="checkbox"
          checked={marketingOptIn}
          onChange={toggle}
          disabled={saving}
          className="h-4 w-4 rounded"
        />
        <span className="text-sm">{marketingOptIn ? 'Subscribed to offers' : 'Not subscribed to offers'}</span>
      </label>
    </div>
  )
}

# WhatsApp Marketing Preferences

## Overview

Customers control whether they receive promotional WhatsApp messages via:
- The customer PWA (profile page toggle)
- The WhatsApp bot (STOP OFFERS / START OFFERS keywords)
- Admin override (customer detail page)

Lifecycle/utility messages (booking confirmations, reminders, etc.) are not
affected by the marketing preference — only MARKETING-category templates.

## Template Categories

See `lib/messaging-templates.ts`. Each template has `category: 'UTILITY' | 'MARKETING'`.

**MARKETING templates** (require opt-in):
- `booking_cancelled`
- `quote_ready`
- `slot_available`
- `job_offer`
- `technician_welcome`

**UTILITY templates** (sent by default, only blocked if service opt-in is false):
- All job lifecycle templates (booking_confirmation, provider_on_the_way, etc.)

## Data Model

`Customer` fields:
| Field | Default | Meaning |
|-------|---------|---------|
| `whatsappServiceOptIn` | `true` | May receive utility/lifecycle msgs |
| `whatsappMarketingOptIn` | `false` | May receive promotional msgs |
| `whatsappMarketingOptInAt` | null | When they opted in |
| `whatsappMarketingOptOutAt` | null | When they last opted out |
| `whatsappMarketingSource` | null | Last change source: bot/pwa/admin/webhook/import |
| `lastWhatsappPrefSyncAt` | null | Last time pref was updated |

Every change is written to `WhatsappPreferenceLog` with actor, source, and before/after values.

## Enforcement

`lib/whatsapp-policy.ts` exports `canSend(phone, templateName) → PolicyResult`.
Every function in `lib/whatsapp.ts` calls this before sending.

## Opt-Out Flows

### Via Bot
Send any of: `stop offers`, `unsubscribe`, `stop marketing`, `no marketing`, `opt out`
→ Bot replies with confirmation. Re-subscribe with `start offers` or `subscribe`.

### Via PWA
Profile page → "WhatsApp Notifications" card → uncheck "Subscribed to offers".

### Via Admin
Customer detail page → "WhatsApp Preferences" section → server action to toggle.

## Adding New Templates

1. Add entry to `TEMPLATE_REGISTRY` in `lib/messaging-templates.ts` with correct `category`
2. Add send function in `lib/whatsapp.ts` using the `canSend()` guard pattern
3. `canSend()` will enforce the correct preference automatically
```
