# Provider credits language and pricing

## Product decision

Plug A Pro provider credits are prepaid platform units, not financial credit. User-facing copy must use credits terminology and must not describe Plug A Pro as offering loans, advances, credit checks, a credit facility, or a credit application.

## Pricing

1 credit = R50.

Provider top-up package displays and credit issuance must derive from the shared provider credit price constant instead of hardcoded Rands-per-credit copy.

## Consumption rule

Credits are consumed only when a customer selects a provider and that provider accepts the selected job.

No credits are used for:

- previewing a lead
- showing interest
- being shortlisted
- customer selection before provider final acceptance
- declining
- expiry

## User-facing wording

Use:

- Provider Credits
- credits balance
- available credits
- credits history
- provider credits terms and rules
- View credits rules
- View credits history
- Buy credits
- Top up credits

Avoid the following terms in all user-facing copy:

| Banned term | Use instead |
|---|---|
| credit policy | credits rules / credits terms |
| View credit policy | View credits rules |
| credit balance | credits balance / available credits |
| credit history | credits history |
| credit facility | (do not use — Plug A Pro does not offer a credit facility) |
| credit check | (do not use — Plug A Pro does not run credit bureau checks) |
| loan | (do not use) |
| advance | (do not use) |
| financing | (do not use) |
| apply for credit | (do not use — providers buy credits, not apply for them) |
