# Plug A Pro PWA — UI Audit & Modernization (2026-05-02)

## Summary

Full UX/UI pass across the customer and provider PWAs. Direction: premium dark "Black Apple"-inspired system with shared tokens, components, and operational clarity. Every change is centralised on tokens and shared primitives — no per-screen one-offs.

## What was already in place

- **shadcn primitives** (18 components) wired through CSS custom properties.
- **`tone-*` semantic system** in `globals.css` for neutral / info / success / warning / danger / brand.
- **`StatusBadge`** with full enum coverage for Job, Booking, Match, JobRequest, Quote.
- **PWA manifest**, safe-area handling, 44 × 44 minimum touch target, `MobileGate`.
- **Bottom nav** layout chain on customer + provider routes.

## What was wrong

| Area | Issue |
|---|---|
| Theme | Default theme was `light` with a blue / pink radial gradient body — fights any "premium dark" direction. |
| Provider screens | ~27 hardcoded utility-color callouts (`bg-emerald-50`, `bg-amber-50`, `bg-blue-50`, `text-green-600`) instead of `tone-*` classes. |
| Provider lead detail | 13 inline coloured `<div>` callouts repeated the same shape. |
| Customer booking detail | Inline `bg-orange-50 dark:bg-orange-900/10` and `bg-emerald-50` alert boxes that didn't theme cleanly. |
| AttachmentThumbnail | Light-only `border-amber-200 bg-amber-50 text-amber-900` fallback. |
| ServiceAreaPicker / SkillPicker | `border-gray-300` light-mode escape on checkboxes. |
| EarningsDashboard | `text-green-600 dark:text-green-400` hardcoded for the "Paid" status. |
| Provider job detail | `text-blue-500` underline on the Google Maps link instead of `text-primary`. |
| Bottom nav | Active state communicated via colour only (a11y issue). |
| Bookings list | No skeleton on initial load. |
| Provider home | Inline 4-up `grid-cols-2` stat boxes; not reusable. |
| Provider home | Profile-completeness rendered as plain text + percentage; no progress bar. |
| Manifest | `theme_color #2563eb`, `background_color #f4f7fb` — light-era values. |

## What we changed

### Foundation
- `globals.css` — refreshed dark tokens (near-black `#050608` background, `#0d0e12` cards, soft translucent borders, refined shadows, restrained primary `#4a8cff`). Removed body radial gradients. Added `--surface-elevated`, `--shadow-pop`, `--radius-{sm,md,lg,xl,2xl}`.
- `app/layout.tsx` — `defaultTheme="dark"` + `forcedTheme="dark"`, viewport `colorScheme: 'dark light'`, `viewportFit: 'cover'`, `appleWebApp.statusBarStyle: 'black-translucent'`, dark class hard-set on `<html>` so first paint matches.
- `manifest.json` — `theme_color`, `background_color` aligned with `--background`.

### New shared components
All under `field-service/components/shared/`:

`PageHeader`, `ActionBar`, `EmptyState`, `ErrorState`, `LoadingSkeleton` (+ `CardSkeleton` / `ListSkeleton` / `StatGridSkeleton`), `StatCard`, `AlertCallout`, `MoneyInput`, `FormField`, `CompletionMeter`, `ProviderCard`, `JobCard`. All take their visuals from CSS variables.

### Hardcoded-colour sweep (zero remaining in customer + provider screens)
- Replaced 13 callout boxes in `provider/leads/[leadId]/page.tsx` with `<AlertCallout>`.
- Replaced 5 callouts in `provider/credits/page.tsx`, `provider/availability/page.tsx`, `provider/quotes/[matchId]/page.tsx` with `<AlertCallout>`.
- Replaced light-only fallback in `AttachmentThumbnail` with `tone-warning`.
- Swapped `text-green-600 dark:text-green-400` for `text-[var(--tone-success-fg)]` in `EarningsDashboard`.
- Swapped `border-gray-300` for `border-input` in `ServiceAreaPicker` + `SkillPicker` + `provider/profile`.
- Swapped `text-blue-500 underline` for `text-primary underline-offset-4 hover:underline` in provider job detail.
- Swapped customer bookings timeline `border-green-500 bg-green-500` for `border-primary bg-primary`.

### Provider dashboard rebuild
`provider/page.tsx` now uses `PageHeader` + 4× `StatCard` + `CompletionMeter` + `AlertCallout` for selected-pending and low-credit nudges + the new shared `JobCard`.

### Loading skeletons
`loading.tsx` files added at:
- `app/(customer)/bookings/loading.tsx`
- `app/(provider)/provider/loading.tsx`
- `app/(provider)/provider/leads/loading.tsx`

### Accessibility polish
- Bottom-nav active state now also shows a top dot (`app-nav-link.tsx`) so the cue isn't colour-only.
- `FormField` wires `aria-describedby`, `aria-invalid`, `aria-required`.
- `AlertCallout` uses `role="alert"` for danger / warning, `role="status"` otherwise.
- `EmptyState` and `ErrorState` use semantic roles + visible focus rings on actions.
- `viewportFit: 'cover'` so safe-area insets work on notched devices.
- `prefers-reduced-motion` block in `globals.css`.

## What was deliberately not changed

- WhatsApp deep-link routes and tokens — preserved exactly.
- Server actions, Prisma queries, payment/credit flow logic.
- shadcn primitive APIs — extended via tokens, not re-implemented.
- Light theme — kept as fallback only; dark is forced via `ThemeProvider`. We can revisit if a customer theme toggle becomes a product requirement.
